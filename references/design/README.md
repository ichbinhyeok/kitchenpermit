# Hood Design References

This folder stores the locked visual references for the hood MVP.

These are not implementation files.
They are design-direction references used to keep taste, hierarchy, density, and page structure stable during implementation.

## Structure

- [primary](C:/Development/Owner/hood/references/design/primary)
  - the main visual direction
- [secondary](C:/Development/Owner/hood/references/design/secondary)
  - supporting references for cleaner page rhythm and form structure
- [rejected](C:/Development/Owner/hood/references/design/rejected)
  - examples of directions we explicitly do not want as the main product language

## Usage rules

- Use these references for tone, layout rhythm, panel hierarchy, spacing, and information structure.
- Do not reuse their copy, labels, or pricing language verbatim.
- Do not inherit dashboard, billing, analytics, portal, or enterprise-platform implications from them.
- Rejected references are kept to clarify what to avoid, not what to imitate.

The canonical rules still live in [08_design_direction.md](C:/Development/Owner/hood/spec/08_design_direction.md).
